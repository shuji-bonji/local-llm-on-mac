## なぜローカル LLM を「建てる」のか

クラウドの LLM はすでに十分に賢く、API を叩けば動きます。それでも手元の Mac に LLM サーバを建てる狙いは、**インフラ運用の習得そのものではありません**。目的は、**特定のベンダーやツールに依存しない「選定眼」と「設計判断力」を、自分の手で原理から積み上げること**、そして**クラウドに出せないデータと処理を、自分の管理下で AI にかけられるようにすること**です。

クラウド越しに使っているだけだと、「なぜそのモデルなのか」「なぜそのコンテキスト長なのか」「1回の応答の実コストは何か」が**ブラックボックスのまま**になりがちです。手元で建てると、モデルサイズとメモリの綱引き、コンテキスト窓の予算、tool calling が落ちる境界——といった**普段は見えない制約**が、すべて自分の計器に映ります。

## ローカルだからできること（メリットと使いどころ）

ローカルに置くと、クラウド API では難しい使い方が開けます。

**得られるメリット**:

- **秘匿データを外に出さない** — 契約書・個人情報・社内コードといった機密を、手元で完結して AI にかけられる（プライバシー・コンプライアンス）。
- **従量課金もレート制限もない** — 大量バッチ処理や試行錯誤を、コストやスロットリングを気にせず回せる。
- **再現性を自分で握れる** — モデルとバージョンを固定でき、提供側の都合で挙動が勝手に変わらない。
- **ネット非依存** — LAN 内・オフラインで動く。

**具体的なユースケース**（いずれも本書で実際に作る／繋ぐ）:

| ユースケース                   | 中身                                                              | 本書の該当部                       |
| ------------------------------ | ----------------------------------------------------------------- | ---------------------------------- |
| **翻訳専門 LLM エージェント**  | docx の書式を保ったまま翻訳し、品質を自動ゲートして再翻訳まで回す | 第5部                              |
| **秘匿データを使った AI 活用** | 手元の文書やコードベースを外に送らず検索・要約・QA                | 第3部（`@codebase`）/ 第4部（MCP） |
| **コーディング補助**           | VS Code で Copilot を代替（FIM 補完・コード Chat）                | 第3部                              |
| **領域特化エージェント**       | 自作 MCP（法令・会計など）のツール群を呼ぶエージェント            | 第4部                              |

そして、こうして**建てて・繋いで・運用する過程そのもの**が、「自分や業務に合わせて LLM を育てていく」ノウハウの蓄積になります。どのタスクにどのモデルを当てるか、システムプロンプトや文脈・記憶をどう設計するか、どんなツール（MCP）を持たせ、結果をどう評価して改善するか——これらは**ローカル／クラウドを問わず通用する運用知**で、クラウド LLM を使うときにもそのまま効きます。ローカル環境は、課金やレート制限を気にせずこの「育てる」試行錯誤を回せる土俵になります。

本書を読み終えたとき、あなたの手元には次が残ります。

- ブラウザから使えるローカル LLM サーバ（OpenAI 互換 API つき）
- VS Code やエージェントから叩けるゲートウェイ
- 「どのモデルを・どんな設定で・どう繋ぐか」を**自分の計測に基づいて判断する目**
- 自分／業務に合わせて LLM を**育てていく運用ノウハウ**（ローカル・クラウドを問わず通用する）

## 対象読者と前提環境

**対象読者**:

- 手元の Mac でローカル LLM を**実際に建てて運用したい**人
- クラウドに出せない**秘匿データを AI で活用したい**人
- クラウド LLM をブラックボックスのまま使うのではなく、**選定と設計の判断軸を持ちたい**人
- TypeScript / シェル / SSH が読める程度の開発者（深い ML 知識は不要）

**前提環境**:

| 項目         | 想定                                                                                       |
| ------------ | ------------------------------------------------------------------------------------------ |
| マシン       | Apple Silicon Mac（本書の実機は **M1 Pro / 32GB**。16GB でも軽量モデルなら可）             |
| OS           | macOS（ターミナル / SSH / Homebrew が使える状態）                                          |
| クライアント | 同一 LAN 上の別 Mac（任意。ヘッドレス運用の章で使用）                                      |
| 方針         | **Docker は使わない**（pip / Homebrew / LaunchDaemon を優先し、VM オーバーヘッドを避ける） |

:::message
本書のサーバ機にはハンズオン上の固有名（ホスト名 `neko8` など）が出てきますが、いずれも**読み替え用の具体例**です。ご自身の環境名に置き換えて読んでください。
:::
